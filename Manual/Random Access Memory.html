On the Subject of Random Access Memory

Now the bomb runs much faster!

This module contains a screen, which includes a progress bar (which indicates the current memory usage of the bomb), memory and a clear memory button.

Clear memory before the memory become full, which will freeze the bomb, forcing the bomb to reset and give a strike.

After a certain amount of module is solved, the RAM might get faulty and the module will solve itself.

The plan

How the module will work:

  • The RAM limit will be generated between 100-999
  • The unit will be randomly generated (the more variant the better).
  • The initial RAM value will be 40%*.
  • The memory will increase by 5-12%* every 4-10s* (randomized)
  • If the memory is full, freeze the bomb for 2s* before giving a strike.
  • The clear memory button clears the memory (obviously) to 40-50%*.
  • After clear memory button is pressed, there should be a delay before refreshing. (now it's instant)
  • If possible, there should be also a prompt on how many "applications" are closed, depending on current percentage, while being RNGed.
  • The module will solve after 50%-66%* of the modules are solved.
